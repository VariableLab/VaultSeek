# VaultSeek 压力测试方案

## 测试目标
验证 VaultSeek 在高负载场景下的性能表现、稳定性和资源占用情况。

## 测试环境
- **CPU**: Apple M系列 / Intel/AMD 现代多核处理器
- **内存**: 16GB+ 推荐
- **存储**: SSD
- **操作系统**: macOS / Windows / Linux
- **Rust版本**: 1.75+
- **模型**: BGE-Small-ZH (ONNX, ~95MB, 384维向量)

---

## 测试场景与指标

### 1. 索引构建压力测试

| 场景 | 文档数量 | 平均文档大小 | 总数据量 | 预期指标 |
|-----|---------|-------------|---------|---------|
| 轻量级 | 100 | 50KB | 5MB | < 30秒 |
| 中等级 | 1,000 | 100KB | 100MB | < 5分钟 |
| 重度级 | 5,000 | 200KB | 1GB | < 30分钟 |
| 极限级 | 10,000 | 500KB | 5GB | < 60分钟 |

**关键指标**:
- 索引构建吞吐量 (docs/sec, chunks/sec)
- 内存峰值占用
- CPU 利用率
- SQLite 数据库增长大小
- 向量化成功率

### 2. 搜索性能压力测试

| 场景 | 向量总数 | 并发查询数 | 查询类型 | 预期指标 |
|-----|---------|-----------|---------|---------|
| 单用户轻量 | 1,000 | 1 | 语义+关键词 | < 100ms |
| 单用户中等 | 10,000 | 1 | 语义+关键词 | < 500ms |
| 单用户重度 | 50,000 | 1 | 语义+关键词 | < 2s |
| 并发压力 | 10,000 | 10 | 混合 | P99 < 1s |

**关键指标**:
- 平均查询延迟 (P50, P95, P99)
- QPS (Queries Per Second)
- 并发下的延迟抖动
- 结果准确性 (Recall@K)

### 3. 增量索引/实时监控测试

| 场景 | 操作 | 频率 | 预期指标 |
|-----|------|------|---------|
| 文件新增 | 单文件 | 10次/分钟 | < 2秒感知到变化 |
| 文件修改 | 单文件 | 10次/分钟 | < 2秒更新索引 |
| 文件删除 | 单文件 | 10次/分钟 | < 1秒清理索引 |
| 批量操作 | 100文件 | 1次 | < 30秒完成 |

### 4. 内存与资源压力测试

| 指标 | 正常阈值 | 警告阈值 | 危险阈值 |
|-----|---------|---------|---------|
| 内存占用 (空闲) | < 200MB | 200-500MB | > 500MB |
| 内存占用 (索引中) | < 1GB | 1-2GB | > 2GB |
| 内存占用 (搜索中) | < 300MB | 300-800MB | > 800MB |
| CPU 空闲 | < 5% | 5-20% | > 20% |
| CPU 索引中 | 60-90% | - | 持续 100% |
| 数据库大小 | 向量数 * 2KB | - | > 10GB |

### 5. 长时间运行稳定性测试

| 测试时长 | 操作模式 | 成功标准 |
|---------|---------|---------|
| 1小时 | 持续索引+搜索 | 无内存泄漏, 无崩溃 |
| 8小时 | 模拟真实工作负载 | 内存增长 < 10%, 无错误 |
| 24小时 | 后台运行+定期操作 | 服务可用性 100% |

### 文档生成脚本需求
- 支持生成: Markdown, PDF, DOCX, TXT, XLSX
- 可控制: 文件数量、大小分布、内容相似度
- 内容类型: 技术文档、会议记录、代码片段、表格数据

---

## 故障注入测试

| 故障类型 | 注入方式 | 预期行为 |
|---------|---------|---------|
| 磁盘满 | 模拟磁盘空间不足 | 优雅报错, 不丢失已有数据 |
| 权限错误 | 只读目录/文件 | 跳过报错, 继续处理其他文件 |
| 网络中断 | 断开 API 连接 | 本地功能正常, 云端功能降级 |
| 模型加载失败 | 损坏 model.onnx | 启动报错, 明确提示 |
| 数据库损坏 | 破坏 SQLite 文件 | 重建索引选项 |
