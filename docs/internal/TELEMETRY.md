# 隐私、合规与遥测策略 (TELEMETRY.md)

本文件规定了 VaultSeek 如何处理与用户数据相关的数据收集行为，确保商业化运营合规。

## 1. 隐私原则 (The Privacy Pledge)
- **数据本地化**：所有文档原文、向量索引、对话上下文，均存储在用户本地 SQLite 中。我们（开发团队）**永远无法**通过任何手段远程访问这些数据。
- **透明审计**：任何网络行为（如连接 API）必须在 UI 上有明确的状态指示。

## 2. 遥测策略 (Telemetry)
- **允许收集 (白名单)**：
    - **崩溃报告**：当发生未捕获 Panic 时，发送匿名崩溃堆栈信息 (使用 `sentry` 或类似工具)，包含硬件架构 (M1/x64) 和版本号。
    - **使用指标 (匿名)**：仅记录“关联了多少次知识库”、“平均处理文档耗时”，目的是为了优化性能，**绝不记录查询内容**。
- **严禁收集 (黑名单)**：
    - 用户搜索的内容。
    - 用户关联的文件夹本地物理路径。
    - 大模型的 API Key 或本地模型名称。

## 3. 合规性审查
在发布每一版更新前，必须通过 `grep` 扫描 codebase，确认没有在代码中硬编码任何敏感的日志输出语句 (`println!`)。
