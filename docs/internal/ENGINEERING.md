# VaultSeek 研发准则与工程规范 (Engineering Standards)

**最后更新**: 2026-06
**适用对象**: 核心开发团队

---

## 1. 核心代码规范

### 1.1 Rust 后端规范 (Zero Tolerance)
*   **意图解析层 (Intent Layer)**：必须作为 RAG 链路的第一站。严禁在未经查询改写 (Query Expansion) 的情况下直接执行检索。
*   **绝对禁止的 Error 处理**：核心业务严禁 `unwrap()`。必须通过 `Result` 显式抛出错误，并统一映射为前端可见的逻辑报错。
*   **并发控制**：所有文档解析必须利用 `rayon` 实现多核处理；所有流式请求必须利用 `tokio` 异步执行。

### 1.2 React 前端规范
*   **交互原则**：基于“主动引导”设计，当用户输入模棱两可时，前端界面必须提供明确的分类引导（而非直接抛出错误）。
*   **数据流向**：前端状态管理必须清晰区分 `UserQuery`, `RetrievedContext`, `ModelResponse`，确保右侧面板能实时映射对应的 Token 来源。
