# VaultSeek 上线发布检查清单 (Release Checklist)

在任何版本被标记为 `v*.*.*` 并推送至 `main` 分支触发 GitHub Actions 之前，主程序员必须完成以下 20 项硬性指标的检查。

---

## 🔒 1. 核心安全与隐私防线 (Privacy & Security)
- [ ] **数据库隔离确认**：确认 `.gitignore` 包含了所有 `*.db`, `*.db-shm`, `*.db-wal` 记录。绝对没有任何测试用户的 SQLite 缓存文件被提交。
- [ ] **网络纯净度测试**：在断网环境下启动软件，除了设定的云端大模型 API 请求外，没有任何其他隐蔽的网络外发请求（如 Telemetry, Tracker）。
- [ ] **密钥脱敏**：前端 `App.jsx` 和后端代码中，默认的 API Key 必须为空或指向 `.env`（正式上线前必须清理掉我们在测试阶段留下的明文 Kimi Key）。

## ⚙️ 2. 性能与资源指标 (Performance Benchmark)
- [ ] **冷启动速度**：在 M1 Mac 上，从点击图标到界面渲染完毕（包括 ONNX 引擎初始化）必须 < 1.5 秒。
- [ ] **内存水位限制**：索引 1000 个 Markdown 文件时，Tauri 进程加上后台 Rust 解析进程的内存总占用不得超过 300MB。
- [ ] **检索延迟**：1000 个切块规模下，混合检索（向量 + BM25）耗时 < 100ms。

## 🖥️ 3. UI 与交互底线 (UX Must-Haves)
- [ ] **Spotlight 验证**：`Alt + S` 必须能在任意全屏应用上唤起 VaultSeek，且**搜索框必须自动获得焦点**，光标闪烁。
- [ ] **缩放保护**：拖拽窗口边缘缩小到极致时，左侧边栏和右侧面板没有消失或文字重叠（CSS Flex 约束有效）。
- [ ] **防抖控制**：连续疯狂点击“发送”按钮，不会导致后端发出多个重复的模型流式请求。

## 🧠 4. RAG 与模型稳定性 (RAG Integrity)
- [ ] **幻觉熔断**：输入与本地库完全无关的随机字符，系统必须触发熔断，回复“本地知识库中未找到相关内容”，而不是大模型瞎编。
- [ ] **来源穿透**：右侧展示的引用片段，点击“打开原文”必须能精确呼出对应的系统默认阅读器（或 Obsidian）。
- [ ] **流式断流恢复**：在模型回答到一半时人为切断网络，UI 必须正常捕捉错误，而不是永远处于“正在生成”的死锁状态。

## 📦 5. 跨平台兼容性 (Cross-Platform Builds)
- [ ] macOS (Apple Silicon)：确认 `tauri.conf.json` 中配置了正确的图标和架构。
- [ ] Windows (x64)：确认没有使用仅限 Unix 系统的 API (如强依赖 `sh` 脚本)。

## 4. 跨库检索与合成功能 (Phase 4 验证)
- [x] **批量并发读写**：BatchProcessor 已通过单元测试，验证了跨库异步处理能力。
- [x] **内存水位优化**：使用了 `tokio::spawn` 异步任务隔离，避免一次性加载过多文档。
- [x] **前端交互闭环**：实现了 BatchExportModal，支持知识库选择与进度感知。
