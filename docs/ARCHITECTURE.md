# VaultSeek 架构设计 (v1.1 稳定版)

## 1. 核心架构
VaultSeek 采用 **Tauri + Rust** 架构，遵循“本地优先、隐私保障”原则。

- **前端 (React + Tailwind)**: 负责 Spotlight 风格交互、搜索结果展示、索引进度反馈。
- **后端 (Rust)**:
  - **推理引擎 (ort)**: 加载 ONNX 模型进行本地 Embedding。
  - **存储 (SQLite)**: 存储文件元数据 (`files`) 与 语义片段 (`chunks`)。
  - **搜索 (Vector + Keyword)**: 混合检索，支持语义相似度与文件名匹配。

## 2. 数据模型 (v1.1 重构)

### 2.1 数据库结构
- **files**: 存储文档元数据（路径、修改时间、哈希）。
- **chunks**: 存储分块内容、所属文件 ID、片段序号、Embedding 向量（BLOB）。

### 2.2 分块策略
采用滑动窗口算法：
- **Chunk Size**: ~500 字符。
- **Overlap**: ~50 字符。
- **元数据**: 记录片段在原文中的大致偏移量（用于来源定位）。

## 3. 搜索流程
1. 用户输入查询 -> Rust 生成查询向量。
2. 在 `chunks` 表中进行向量相似度计算。
3. 聚合结果，按文件展示，并高亮匹配的片段。
4. 点击结果通过 `open_file` 穿透至本地应用。
