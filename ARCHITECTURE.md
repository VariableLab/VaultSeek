## 1. 系统概述

VaultSeek 是一个本地优先的 AI 语义知识库应用，采用混合架构：本地处理保证隐私和离线能力，云端 One-API 中转站提供高级推理能力。

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **本地优先** | 所有文档解析、向量化、检索在本地完成 |
| **隐私第一** | 文档内容永不出本地，仅推理请求上云 |
| **优雅降级** | 云端不可用时，本地搜索仍可用 |
| **成本可控** | 每日 10 次免费额度，超出后引导付费 |

## 3. 核心模块设计

### 3.1 文档导入模块

**职责**: 解析 PDF/Word/TXT 文件，生成纯文本

**技术选型**:
- PDF: `pdf-parse` (Node.js)
- Word: `mammoth` 或 `docx` (Node.js)
- TXT: 原生读取

**处理流程**:
```
用户选择文件
    │
    ▼
格式校验 (PDF/Word/TXT)
    │
    ▼
文件去重检查 (基于文件 hash)
    │
    ▼
解析为纯文本
    │
    ▼
智能分块 (按语义/段落/长度)
    │
    ▼
存储到临时队列，等待向量化
```

**错误处理**:
- 文件损坏：提示用户"文件无法解析"
- 格式不支持：提示"仅支持 PDF/Word/TXT"
- 大文件 (>100MB): 分批次处理，显示进度

---

### 3.2 向量化模块

**职责**: 将文本块转换为向量，存储到本地向量库

**技术选型**:
- Embedding 模型：`@xenova/transformers` (支持浏览器/Node.js 的轻量模型)
- 向量库：`chromadb` (本地模式，无需服务器)

**处理流程**:
```
从临时队列取文本块
    │
    ▼
调用本地 Embedding 模型
    │
    ▼
生成向量 (768 维)
    │
    ▼
存储到 Chroma (带元数据：文档 ID、段落位置)
    │
    ▼
更新 SQLite 索引表
```

**性能边界**:
| 配置 | 文档数上限 | 向量化速度 |
|------|-----------|-----------|
| 免费版 | 50 文档 | 实时处理 |
| 专业版 | 500 文档 | 实时处理 |
| 企业版 | 5000 文档 | 批处理 + 进度条 |

---

### 3.3 本地检索模块

**职责**: 用户提问时，从本地向量库检索相关文档

**流程**:
```
用户提问
    │
    ▼
调用本地 Embedding 模型生成问题向量
    │
    ▼
Chroma 相似度检索 (Top-K)
    │
    ▼
返回相关文档片段 + 来源位置
    │
    ▼
判断是否需要 LLM 总结
    │
    ├── 否 ──► 直接返回原始片段
    │
    └── 是 ──► 进入 LLM 流程
```

---

### 3.4 LLM 服务模块 (One-API)

**职责**: 管理云端推理请求和额度统计

**配置结构**:
```typescript
interface OneAPIConfig {
  // 模式：official(官方额度) | custom(自带 Key)
  mode: 'official' | 'custom';
  
  // 官方模式配置
  official?: {
    endpoint: string;  // One-API 服务端地址
    apiKey: string;    // 用户的 API Key (可选，默认用官方)
    model: string;     // 默认模型
  };
  
  // 自带 Key 模式配置
  custom?: {
    endpoint: string;
    apiKey: string;
    model: string;
  };
  
  // 额度统计
  usage: {
    todayCount: number;   // 今日已用次数
    lastReset: string;    // 上次重置时间
    dailyLimit: number;   // 每日限额 (默认 10)
  };
}
```

**额度检查流程**:
```
LLM 请求
    │
    ▼
检查今日剩余额度
    │
    ├── 有剩余额度 ──► 调用 One-API
    │                  │
    │                  ▼
    │             更新额度统计
    │                  │
    │                  ▼
    │             返回推理结果
    │
    └── 额度用完 ──► 弹窗引导付费
                     │
                     ├── 充值额度 (198 终身)
                     │
                     ├── 绑定自带 Key
                     │
                     └── 稍后再说 (返回本地模式)
```

### 3.5 存储模块

**SQLite 表结构**:

```sql
-- 文档元数据表
CREATE TABLE documents (
  id TEXT PRIMARY KEY,
  filename TEXT,
  filetype TEXT,
  filesize INTEGER,
  import_time DATETIME,
  chunk_count INTEGER,
  status TEXT  -- 'pending' | 'processing' | 'completed' | 'error'
);

-- 向量索引表
CREATE TABLE vector_index (
  id TEXT PRIMARY KEY,
  document_id TEXT,
  chunk_index INTEGER,
  chunk_text TEXT,
  chunk_embedding BLOB,  -- Chroma 内部管理
  FOREIGN KEY (document_id) REFERENCES documents(id)
);

-- 用户配置表
CREATE TABLE user_config (
  key TEXT PRIMARY KEY,
  value TEXT  -- JSON 字符串
);

-- 使用统计
CREATE TABLE usage_stats (
  date TEXT PRIMARY KEY,
  llm_count INTEGER DEFAULT 0,
  search_count INTEGER DEFAULT 0
);
```

## 6. 安全与隐私

### 6.1 数据边界

| 数据类型 | 存储位置 | 是否上云 |
|---------|---------|---------|
| 文档原文 | 本地 | ❌ |
| 向量索引 | 本地 | ❌ |
| 用户配置 | 本地 | ❌ |
| LLM 问题 | 临时上云 (仅推理请求) | ✅ |
| LLM 回答 | 本地缓存 | ✅ (来源云端) |

### 6.2 API Key 管理

- 官方模式：Key 存储在本地 SQLite，加密存储
- 自带 Key 模式：用户自行管理，应用不保存
